Build a production-style MVP web app called “UrgentFix”.

APP SUMMARY
UrgentFix is an on-demand emergency home repair platform. Customers can request urgent help for plumbing, electrical, locksmith, and boiler/heating issues. Providers can accept jobs, update status, and complete work. Admins can verify providers, monitor bookings, and manage disputes.

GOAL
Create a clean, modern, mobile-friendly MVP that works end-to-end with:
1. Customer booking flow
2. Provider job acceptance flow
3. Admin verification/management flow
4. Real-time booking status updates
5. Stripe-ready payment structure
6. Clean database structure and scalable code organization

IMPORTANT BUILD RULES
- Build the app as a working MVP, not a landing page mockup
- Use clean, modular, scalable code
- Keep code beginner-readable
- Add comments only where useful
- Avoid fake placeholder logic where real logic can be implemented
- Use deterministic state handling for bookings
- Use form validation
- Include error handling for all major actions
- Make the UI feel like a real startup product
- Use a professional modern design with strong UX clarity
- Prioritize mobile-first layout
- Do not overcomplicate version 1 with unnecessary features

RECOMMENDED STACK
Use this stack unless Replit suggests a better fully compatible option:
- Frontend: React + TypeScript + Vite
- Styling: Tailwind CSS
- UI components: shadcn/ui style components or equivalent clean component system
- Backend: Node.js + Express or Replit-compatible server routes
- Database: PostgreSQL with Prisma ORM
- Auth: email/password auth with role-based access
- State: simple React state or lightweight store
- Realtime: polling or lightweight realtime updates
- Payments: Stripe-ready integration structure, even if test mode only
- Maps: leave map integration ready, but do not block MVP if live map key is missing

AUTH + ACCESS RULES
Implement role-based permissions:
- Customers can create and view only their own bookings
- Providers can view only bookings assigned to them or available to them
- Admin can view and manage everything
- Providers cannot mark themselves verified
- Customers cannot access admin/provider management pages
- Admin can verify/suspend providers

- Select issue type

- Add description

- Select urgency:

  - Today
  - Scheduled

- Show estimated callout fee
- Confirm request

- Booking confirmation page
